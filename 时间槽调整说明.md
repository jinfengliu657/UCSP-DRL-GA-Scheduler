# 时间槽调整说明

## 修改内容

根据实际需求，将时间槽从 **6节课制** 调整为 **5时段制**：

### 原设置（6节课制）
- 时间槽: 0-29 (5天 × 6节课)
- 每天6节课，连续编号

### 新设置（5时段制）
- 时间槽: 0-24 (5天 × 5时段)
- 每时段代表2节课

### 时段划分
| 时段编号 | 时间段 | 课程数 |
|---------|--------|--------|
| 时段1 (0) | 上午第1时段 | 2节课 |
| 时段2 (1) | 上午第2时段 | 2节课 |
| 时段3 (2) | 下午第1时段 | 2节课 |
| 时段4 (3) | 下午第2时段 | 2节课 |
| 时段5 (4) | 晚上时段 | 2节课 |

### 槽位示例
| 槽位范围 | 星期 | 包含时段 |
|---------|------|----------|
| 0-4 | 周一 | 上午1-2、下午1-2、晚上1 |
| 5-9 | 周二 | 上午1-2、下午1-2、晚上1 |
| 10-14 | 周三 | 上午1-2、下午1-2、晚上1 |
| 15-19 | 周四 | 上午1-2、下午1-2、晚上1 |
| 20-24 | 周五 | 上午1-2、下午1-2、晚上1 |

## 修改的文件

1. **README.md** - 更新编码方案说明
2. **algorithms/constraint_solver.py** - num_slots = 25
3. **models/chromosome.py** - 所有 range(30) 改为 range(25)
4. **data/data_loader.py** - 更新时间槽字符串转换
5. **evaluation/fitness.py** - 更新天数和时段计算
6. **run_experiment.py** - 更新软约束指标计算

## 代码修改要点

### 天数计算
```python
# 原来: day = slot // 6
# 现在: day = slot // 5
```

### 时段计算
```python
# 原来: period = slot % 6
# 现在: period = slot % 5
```

### 上午/下午判断
```python
# 原来:
if period < 3:  # 上午
    ...
else:  # 下午
    ...

# 现在:
if period < 2:  # 上午 (时段0-1)
    ...
elif period < 4:  # 下午 (时段2-3)
    ...
# 时段4为晚上，不计入上午/下午集中度
```

## 注意事项

1. 数据库中的固定时间课程需要使用1-25的编号（代码中会自动转换为0-24）
2. 适应度函数中的教学楼集中度只考虑上午和下午，晚上时段不计入
3. 所有涉及时间槽遍历的地方都已更新为25个槽位
